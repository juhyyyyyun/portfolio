문제 (Trouble)
불필요한 DB 메타데이터 노출로 인한 보안 리스크, 엔티티 간 양방향 연관 관계로 인한 JSON 순환 참조 에러 발생, 화면 요구사항 변경 시 DB 구조까지 영향을 받는 높은 결합도 문제 직면.
A
해행 (Action)
엔티티와 UI 레이어의 관심사를 분리하기 위해 DTO(Data Transfer Object) 패턴을 전면 도입하고, 화면에 꼭 필요한 데이터만 정제하여 반환하도록 로직을 리팩토링함.
R
결과 (Result)
